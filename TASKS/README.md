## Estrutura

```
TASKS/
  SEGURANÇA/
    backlog/   ← tarefas pendentes
    done/      ← tarefas concluídas
  UX/
    backlog/
    done/
  PERFORMANCE/
    backlog/
    done/
```

## Convenção de nomes

`[PRIORIDADE]-[SEVERIDADE]_[slug].md`

- Prioridade: `P1` (esta semana) · `P2` (próximo sprint) · `P3` (backlog)
- Severidade: `CRITICO` · `ALTO` · `MEDIO` · `BAIXO`

## Workflow

1. Task começa em `backlog/`
2. Quando iniciada, adicionar `## Em progresso` ao ficheiro
3. Quando concluída, mover para `done/` e preencher `## Resolução`
