# ADR-002 — Usar React + TypeScript + Vite como frontend

**Data:** 2026-06-11  
**Estado:** Aceite

---

## Contexto

Necessidade de uma SPA moderna, type-safe, com ecossistema rico para construir interfaces de gestão.

## Decisão

Usar React 18 + TypeScript + Vite como stack frontend.

## Consequências

**Positivas:**
- TypeScript previne erros em runtime (crítico para lógica de stock)
- Vite oferece hot reload muito rápido e build otimizado
- React tem ecossistema maduro (componentes, hooks, libraries)
- Tailwind CSS v4 integrado no projeto
- Fácil integração com Supabase SDK

**Negativas:**
- Mais verboso que JavaScript puro
- Curva de aprendizagem TypeScript para devs menos experientes

## Alternativas Consideradas

- **Vue 3 + TypeScript:** Alternativa válida, mas ecossistema menor e menos componentes disponíveis.
- **Next.js:** Adiciona SSR/SSG que não é necessário para sistema interno; complexidade desnecessária.
- **Svelte:** Excelente performance, mas ecossistema menor e menos familiaridade da equipa.
