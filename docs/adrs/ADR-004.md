# ADR-004 — MVP com apenas um utilizador administrador

**Data:** 2026-06-11  
**Estado:** Aceite

---

## Contexto

A ENCIVIL tem um armazém gerido por uma pessoa (encarregado). Implementar multiutilizador no MVP aumentaria significativamente a complexidade (gestão de perfis, permissões granulares, conflitos de acesso).

## Decisão

O MVP suporta apenas 1 utilizador com role `admin`. O utilizador é criado manualmente pelo programador via Supabase Dashboard.

## Consequências

**Positivas:**
- Simplifica muito o MVP: sem gestão de utilizadores, sem perfis de acesso granulares
- RLS mais simples: "autenticado = admin"
- Sem risco de utilizador não autorizado fazer saídas erradas
- Entrega mais rápida

**Negativas:**
- Não permite múltiplos encarregados com rastreabilidade individual
- Se o encarregado estiver ausente, outra pessoa não tem acesso fácil
- Mudança de palavra-passe requer acesso ao Supabase

## Alternativas Consideradas

- **Multiutilizador desde o início:** Correto para o futuro, mas dobra a complexidade do MVP. Reservado para Fase 2.

## Nota

O campo `responsavel` nos movimentos é texto livre, permitindo identificar quem fisicamente executou o movimento mesmo com um único utilizador de sistema.
